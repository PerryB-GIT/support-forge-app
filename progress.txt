## Iteration 1 - US-001

**Completed:** Added CSS animation utilities and hover effects to globals.css

**Learnings:**
- Project uses Tailwind CSS v4 with @import "tailwindcss" syntax
- CSS custom properties already defined for brand colors (--accent-glow, --forge-purple, etc.)
- No dedicated typecheck script - use `npx tsc --noEmit` in apps/web directory
- Added prefers-reduced-motion support for accessibility

**CSS Classes Added:**
- `.card-hover` - lift + glow effect on hover
- `.glow-purple` - purple glow on hover
- `.scale-hover` / `.scale-hover-subtle` - scale effects
- `.animate-fade-up` / `.animate-fade-up-stagger` - scroll reveal
- `.animate-pulse-glow` - pulsing glow effect
- `.animate-float` - floating animation

**Files changed:**
- apps/web/src/app/globals.css

## Iteration 2 - US-002

**Completed:** Added gradient background system to globals.css

**Learnings:**
- CSS pseudo-elements (::before, ::after) work well for overlay effects
- SVG data URIs inline in CSS work for noise textures (keep opacity very low ~0.03)
- Need `pointer-events: none` on overlay pseudo-elements to allow clicks through
- Grid patterns use repeating linear-gradient with background-size for tiling
- Glow orbs use radial-gradient with blur filter for soft light effect

**CSS Classes Added:**
- `.hero-gradient` - main hero linear-gradient with purple spotlight ::before
- `.hero-spotlight-right` - additional right-side radial gradient ::after
- `.section-gradient` - vertical gradient for content sections
- `.surface-gradient` - diagonal gradient for cards/elevated areas
- `.noise-texture` - subtle grain effect using SVG feTurbulence
- `.grid-pattern` - tech aesthetic grid overlay
- `.glow-orb` / `.glow-orb-small/medium/large` - decorative blur effects

**Files changed:**
- apps/web/src/app/globals.css

## Iteration 3 - US-003

**Completed:** Created HeroSection component with image support

**Learnings:**
- Next.js Image component with `fill` prop and `sizes` attribute handles responsive images
- Use `sizes="(max-width: 768px) 800px, (max-width: 1200px) 1200px, 1920px"` for breakpoints
- Layer overlays with z-index: image(0) < spotlights(1) < noise(2) < content(10)
- Inline CSS for radial gradients when classes aren't flexible enough
- Use animation-delay via inline style for staggered animations
- Components can use CSS classes from globals.css directly

**Props created:**
- `title` - main heading (required)
- `subtitle` - description text (optional)
- `backgroundImage` - URL for hero image (optional)
- `gradient` - enable gradient/spotlight effects (default: true)
- `children` - CTA buttons or additional content (optional)
- `size` - "sm" | "md" | "lg" for padding (default: "md")
- `align` - "left" | "center" for text alignment (default: "center")

**Files created:**
- apps/web/src/components/ui/HeroSection.tsx

## Iteration 4 - US-004

**Completed:** Added hero section to Homepage with floating particles

**Learnings:**
- HeroSection title prop should be ReactNode (not string) to support JSX with styled spans
- Floating particle system uses multiple animation keyframes with varied timing for organic feel
- Particles distributed via array math: `left: ${(i * 8.5) % 100}%` for even spread
- Connection lines add network/AI feel using gradient lines with pulse animation
- Wrap HeroSection in outer section for proper positioning of absolute particles container
- Always update prefers-reduced-motion when adding new animations

**CSS Classes Added:**
- `.particles-container` - absolute positioned container for particle system
- `.particle` / `.particle-sm/md/lg` - individual particle nodes with glow
- `.animate-particle-1/2/3` - varied floating animations (8s, 10s, 12s)
- `.particle-line` / `.animate-line-pulse` - connection lines between particles

**Files created:**
- apps/web/src/components/ui/FloatingParticles.tsx

**Files changed:**
- apps/web/src/app/globals.css (particle system CSS)
- apps/web/src/app/page.tsx (HeroSection integration)
- apps/web/src/components/ui/HeroSection.tsx (title prop type fix)
